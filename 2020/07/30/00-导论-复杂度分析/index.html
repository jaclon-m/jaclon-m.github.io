复杂度分析（上）

大O表示法


大O时间复杂度表示法，并不表示代码的具体执行时间，而是表示代码执行时间随数据规模增长的变化趋势，也叫渐进时间复杂度，简称时间复杂度。

二次理解：
其中f(n)是与n相关的函数，表示的是这段代码执行的次数.例如：$f(n)=2n^2 + 2n + 3$
O(n)是在假设代码中每个语句执行时间为unit_time的情况下，代码执行总时间为f(n)*unit_time.

常见的时间复杂量级度分析

分为多项式量级和非多项式量级。
非多项式量级O(2n) 和 O(n!)：又称为NP时间复杂度，执行时间会随着n的增大而急剧增长，不在本次讨论范围内。

O(logn)、O(nlogn)

i=1;
while (i <= n) {
  i = i * 2;
}

这段代码的执行次数$x=log_2n$,所以这段代码的执行时间复杂度为$O(log_2n)$.
我们知道，对数之间是可以互相转换的，$log_3n=log_32*log_2n$.所以对数阶时间复杂度统一表示为$O(logn)$

空间复杂度

时间复杂度的全称是渐进时间复杂度，表示算法的执行时间与数据规模之间的增长关系。空间复杂度全称为渐进空间复杂度，表示算法的存储空间与数据规模之间的增长关系

void print(int n) {
  int i = 0;
  int[] a = new int[n];
  for (i; i <n; ++i) {
    a[i] = i * i;
  }

  for (i = n-1; i >= 0; --i) {
    print out a[i]
  }
}

这段代码空间复杂度为$O(n)$。空间复杂度常见$O(1)、O(n)、O(n2 )$，像 $O(logn)、O(nlogn)$一般用不到.

均摊时间复杂度

均摊时间复杂度：在代码执行的所有复杂度情况中绝大部分是低级别的复杂度，个别情况是高级别复杂度且发生具有时序关系时，可以将个别高级别复杂度均摊到低级别复杂度上。基本上均摊结果就等于低级别复杂度。

例子中的重点

  1. 每个O(n)时间复杂度后面跟的是n-1个O(1)时间复杂度。
  2. n规模增长，对于每个n下面，根据所选元素出现的位置，又有n中情况

二次理解：
只有在输入不同引起代码的复杂度量级不同的情况下，我们才会分析最好、最坏、平均、摊还时间复杂度。在后两种时间复杂度分析的时候，往往还要引入概率。
